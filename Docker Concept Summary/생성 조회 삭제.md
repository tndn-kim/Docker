### image 삭제해보기
/명령어/
- docker image rm (image ID 4자이상)
  - Error response from daemon: conflict: unable to delete dec7a90bd097 (must be forced) - image is being used by stopped container f562952b09ac -> image가 실행되었다 중단된 상태여서 해당 image 종료 못함
  - docker image rm -f (image ID 4자이상) __-f__ 중단된 image 삭제 명령어
- docker image ls -> 리스트 확인
- docker image rm $(docker image -q) 컨테이너에서 사용하고 있지 않은(실행 or 중단) 전체 image 삭제
- docker image rm -f $(docker image -q) 중단된 image 삭제

#### 포그라운드(foreground) , 백그라운드(background)
포그라운드
- 내가 실행시킨 프로그램의 내용이 화면에서 실행되고 출력되는 상태
- 장점: 실시간으로 어떻게 실행되고 어떤 로그들이 생기는지 확인 가능
- 단점: 즉각적으로 신호를 보내야하니 다른 조작이 불가
- #### docker run -> 포그라운드 실행
  
백그라운드
-  내가 실행시킨 프로그램이 컴퓨터 내부적으로 실행되는 상태
-  장점: 프로그램 내부적으로 진행되기 때문에 다른 명령어 추가적으로 입력 가
-  단점: 실시간 진행 상태가 확인 불가능
- #### docker run -d -> 백그라운드 실행

## 컨테이너 조회 중지 삭제

### 컨테이너 조회 명령어
- docker ps -> 실행중인 컨테이너 조회
- docker ps -a -> 중단된 컨테이너를 포함한 모든 컨테이너 조회

### 컨테이너 중지 명령어 -> 미니 컴퓨터 종료
- docker stop ID -> 컨테이너의 정상적인 종료 
- docker kill ID -> 컨테이너 강제 종료 (파일 손상 우려 있음)

### 컨테이너 삭제 명령어
- docker rm ID -> 컨테이너 삭제 (실행중인 컨테이너는 삭제 x)
- docker rm $(docker ps -qa) -> 중지되어있는 모든 컨테이너 삭제
- docker rm -f ID -> 실행중인 컨테이너 삭제
