# *Docker Volume (도커 볼륨)*

### 컨테이너가 가진 문제점
- Docker를 활용한 프로그램 컨테이너로 띄울수 있다.
- 이 프로그램에 업데이트 (기능추가 등등)이 생긴다면 새로운 이미지를 만들어서 컨테이너를 실행
- 이때 Docker 아래와 같이 판단
  - 기존 컨테이너에서 변경된 부분을 수정하 -> 비효율
  - 새로운 컨테이너를 만들어서 통째로 갈아끼우는 방식 -> 효율
- 이런 방식때문에 기존에 내부 데이터도 같이 삭제되는 문제 발생
- 이를 방지하기 위해 볼륨(Volume)이라는 개념을 활용하여 삭제를 방지

## Docker Volume 이란
- 도커 컨테이너에서 데티어를 영속적으로 저장하기 위한 방법
- 컨테이너 자체의 저장공간을 사용하지 않고, 호스트 자체의 저장 공간을 공유해서 사용함  

### *명령어*  
$ docker run -v [호스트의 디렉토리 절대 경로]: [컨테이너의 디렉토리 절대 경로] [이미지명]:[태그명]
- **[호스트의 디렉토리 절대 경로]** 에 디렉토리가 이미 존재할 경우, 호스트의 디렉토리가 컨테이너의 디렉토리를 덮어씌운다
- **[호스트의 디렉토리 절대 경로]** 에 디렉토리가 존재하지 않을경우,. 호스트의 디렉토리 절대 경로에 디렉토리를 새로 만들고 컨테이너의 디렉토리에 있는 파일들을 호스트 디렉토리로 복사해온다

**-e 역할** 환경변수 설정하는 역할
